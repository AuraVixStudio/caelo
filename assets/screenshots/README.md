# Screenshots for the README

Drop your app screenshots here with **these exact filenames** and the README picks them up
automatically. Until they exist, those `<img>` tags show a broken-image icon — so add them
before the README is pushed to the public `main`.

| File | What it should show | Used as |
|---|---|---|
| `hero.png` | The single most impressive view — e.g. **Chat mid-conversation** with live web search + the **Sources/citations** panel, or the **Code** agent with a diff-approval card. Full app window. | Big hero image under the badges |
| `chat.png` | **Chat** module — a live web/X search answer with clickable citations and the cost/token counter. | Gallery (top-left) |
| `code.png` | **Code** module — file tree + editor + the **agent panel** with an approval card / unified diff. | Gallery (top-right) |
| `image.png` | **Image** generation or the **Gallery** with a few generated images. | Gallery (bottom-left) |
| `voice.png` | **Voice** module — Talk/Live or Speak with the cost badge. | Gallery (bottom-right) |

## Recommended capture settings

- **Format:** PNG.
- **Theme:** **dark** (matches the brand; looks best on GitHub's default).
- **Width:** ~1600 px (or 2× for crisp retina). The README scales them down; higher source = sharper.
- **Framing:** capture the app window cleanly. Real content (a real conversation, real generated
  media, a real diff) looks far more professional than empty states — use views that are populated.
- **Privacy:** make sure no API key, token, or private path is visible in the shot.

## Optional extras (not referenced yet, but nice to have)

- `extensions.png` — Extensions → MCP servers / Skills.
- `gallery.png` — the History/Gallery grid.
- A short `demo.gif` of the agent running end-to-end (great hero alternative).

After adding the files, run `git add assets/screenshots && git commit` (or just tell the
assistant — it will verify dimensions and commit everything together).
